Notas

Como desarrolladores de aplicaciones de escritorio, cuando utilizamos librerias GUI solemos utilizar un editor de recursos y vincular el recurso con ID's a nuestro código, para cada componente de un formulario

Posteriormente 'damos vida' a esos controles definidos en recursos y vinculados en codigo fuente, para dotarles de funcionalidad, por ejemplo, para validación, obtención de los datos, etc.

Utilizando el plugin JsonForm (o similares), el paradigma de programación descrito arriba se transforma simplemente en que nuestro script de servidor Harbour, obtiene los datos y genera unas reglas de validación para alimentar el plugin JsonForm

La idea es generar un JSON en servidor, con la estructura adecuada y el plugin nos hará la 'magia' del lado del cliente (en navegador), sin tener que generar codigo HTML, ni scripts de validación

Les invito a que piensen en JSON :)
